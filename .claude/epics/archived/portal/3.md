# Task: ダミーデータ一式 + 型定義 (src/data)

## Description

ai-foo株式会社の設定に基づく型付きダミーデータを `src/data/` に作成する。他のすべてのページタスクがこのデータを import する。

## Acceptance Criteria

- [ ] `src/data/types.ts`: Department / Member / Announcement / Event / FaqItem / QuickLink / CompanyKpi 等の型定義
- [ ] `src/data/company.ts`: 会社概要 + KPI（ARR 12.4億円, 有料契約 860社, NRR 118%, 月次チャーン 0.8% 等サブスクらしい数値）
- [ ] `src/data/departments.ts`: 8 部署（経営企画室6/プロダクト開発部42/プロダクトデザイン部8/セールス部18/カスタマーサクセス部16/マーケティング部10/コーポレート部12/情報システム部6）。各部署にミッション・部長名・人数・使用ツール
- [ ] `src/data/members.ts`: 118 名分の社員（氏名かな付き・部署・役職・入社年・メールアドレス @ai-foo.example.com）。部署人数と整合すること
- [ ] `src/data/announcements.ts`: お知らせ 8 件程度（全社/人事/IT/イベント等のカテゴリ、2026年6-7月の日付）
- [ ] `src/data/events.ts`: 今月（2026年7月）のイベント 5 件程度
- [ ] `src/data/faq.ts`: カテゴリ別 FAQ 12 件程度（経費精算/有給申請/リモート勤務/入退社/IT）
- [ ] `src/data/links.ts`: クイックリンク 8 件程度（Slack/Notion/勤怠/経費精算/GitHub/採用ポータル等、href は `#` のダミー可）

## Technical Details

- すべて TypeScript の named export。実在の人物と被らない自然な日本人名を使用
- 氏名は姓・名バリエーションの組み合わせで 118 名を配列リテラルとして生成してよい
